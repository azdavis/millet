# 5044

<!-- @config language.successor-ml.record-update = true -->

Record update syntax was used, but the thing being updated (the bit before the `where`) wasn't a record.

```sml
val x = 3
val y = { x where b = 5 }
(**     ^^^^^^^^^^^^^^^^^ record update of non-record *)
```

This can happen even when the thing is of partially resolved record type, but not fully resolved.

<!-- @config language.successor-ml.record-update = true -->
<!-- @limit first -->

```sml
fun f x = { x where b = #a x + 1 }
(**       ^^^^^^^^^^^^^^^^^^^^^^^^ record update of non-fully-resolved record *)
```

## To fix

Make sure the thing before the `where` in the record update is a (fully resolved) record.

<!-- @config language.successor-ml.record-update = true -->

```sml
val x = { a = "hi", b = 5, c = false }
val y = { x where a = 5 }
```
